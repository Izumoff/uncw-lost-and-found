# Sprint 3

## User Story:
Security Office Staff can change the status of any report

| Test Case | Input | Expected Output | Actual output |
|---|---|---|---|
| 1 (Success) | **Precondition:** A report exists with status `Pending`. User is logged in as **Security Office Staff**. 1. Open the report detail page 2. Verify the current status is shown | The report detail page opens and shows the current report status | Passed. The report detail page opened successfully and the status was shown on the page as **Pending** |
| 2 (Success) | **Precondition:** User is logged in as **Security Office Staff**. 1. Open any report detail page 2. Verify the staff status-change form is visible | The system shows a status-change form only for staff users | Passed. The staff user could see the status dropdown and **Update Status** button on the report detail page |
| 3 (Failure) | **Precondition:** Status-change form was added to the report detail page. 1. Open the page after adding the template condition for staff check | The page should render without template errors | Failed at first. Django returned template parsing error: `Could not parse the remainder: '(name='Security Office Staff').exists'...` because `.exists()` was used directly in the template. **Fixed.** Staff flag was moved to the view and the template was changed to use `IS_SECURITY_OFFICE_STAFF` |
| 4 (Failure) | **Precondition:** Status dropdown options were added in template. 1. Open the report detail page after adding option selection conditions | The page should render without template errors and the current status should be preselected | Failed at first. Django returned template parsing error: `Could not parse the remainder: '=' from 'report.status='` because the comparison syntax in template was incorrect. **Fixed.** The option conditions were corrected to use `report.status == 'pending'`, `report.status == 'approved'`, and `report.status == 'rejected'` |
| 5 (Success) | **Precondition:** User is logged in as **Security Office Staff** and a report has status `Pending`. 1. Open the report detail page 2. Change status to **Approved** 3. Click **Update Status** | The system updates the report status to **Approved** and reloads the detail page | Passed. The staff user changed the report status successfully, and the detail page reloaded showing the updated status |
| 6 (Success) | **Precondition:** A report exists with status `Pending`. User is logged in as **Security Office Staff**. 1. Change the report status to **Approved** 2. Open `/reports/` | Approved report becomes published and appears in the published reports list | Passed. After the status was changed to **Approved**, the report appeared on `/reports/` |
| 7 (Success) | **Precondition:** A report exists with status `Approved` and is visible on `/reports/`. User is logged in as **Security Office Staff**. 1. Open the report detail page 2. Change status to **Pending** or **Rejected** 3. Open `/reports/` | The report becomes unpublished and no longer appears in the published reports list | Passed. After changing the status from **Approved** to **Pending** or **Rejected**, the report disappeared from `/reports/` |
| 8 (Success) | **Precondition:** A report exists. User is logged in as a **Campus Community User**. 1. Open the report detail page 2. Check whether the staff status-change form is visible | The campus user must not see the staff status-change form | Passed. The report detail page loaded successfully for the campus user and the staff status-change form was not visible |
| 9 (Failure) | **Precondition:** Staff status update flow is implemented. 1. Change a report status as staff 2. Observe the page after redirect | The system should clearly show that the status update was completed | Failed at first. The status value changed in the form, but there was no clear visible feedback that the action had succeeded. **Fixed.** Redirect was updated with `?status_updated=1` and a success alert was added to the detail page |
| 10 (Success) | **Precondition:** Success feedback fix is implemented. User is logged in as **Security Office Staff**. 1. Change a report status 2. Observe the detail page after redirect | The system shows a visible success message and the updated status | Passed. After changing status, the page reloaded with a visible success message: **Report status was updated successfully**, and the new status was shown on the page |
