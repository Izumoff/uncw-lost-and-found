# Sprint 3

## User Story:
Campus Community User can create Found Item report

|---|---|---|---|
| 1 (Success) | **Precondition:** User is logged in. 1. Open `/reports/create/found/` | The system opens the Create Found Item Report page | Passed. The route was added successfully and `/reports/create/found/` opened the Create Found Item Report page |
| 2 (Failure) | **Precondition:** User is not logged in. 1. Open `/reports/create/found/` | The system redirects the user to the login page | Passed. Access was blocked for anonymous user by `login_required` and redirected to login |
| 3 (Success) | **Precondition:** User is logged in and the create page exists. 1. Open `/reports/create/found/` 2. Verify Title field 3. Verify Description field 4. Verify Location field 5. | 4 (Failure) | **Precondition:** User is logged in and the create page is open. 1. Leave **Title** empty 2. Enter Description 3. | 5 (Failure) | **Precondition:** User is logged in and the create page is open. 1. Enter Title 2. Leave **Description** empty 3. | 6 (Success) | **Precondition:** User is logged in and the create page is open. 1. Enter valid Title 2. Enter valid Description 3. Leave Location empty 4. Click **Submit Report** | The system saves the report successfully because Location is optional | Passed. Report was saved successfully with empty Location |
| 7 (Success) | **Precondition:** User is logged in and the create page is open. 1. Enter valid Title 2. Enter valid Description 3. Enter valid Location 4. Click **Submit Report** | The system saves the report and redirects the user after submission | Passed. Report was saved successfully. Initial redirect went to `/reports/` |

Success message became visible |
| 10 (Success) | **Precondition:** User is logged in and has submitted a valid Found Item report. 1. Open saved record in admin or database | The system saves the report with `report_type = "found"` | Passed. Saved report type was `found` |

| 12 (Success) | **Precondition:** User is logged in and has submitted a valid Found Item report. 1. Open saved record in admin or database | The system saves the report with `is_published = False` by default | Passed. Saved report remained unpublished by default |
| 13 (Success) | **Precondition:** User is logged in. 1. Open main navigation 2. Click **Report Found Item** | The system opens the Create Found Item Report page from navigation | Passed. Navigation link was added and opened the create page successfully |
| 14 (Success) | **Precondition:** User is logged in. 1. Open `/reports/create/found/` without `?success=1` | The success message is not displayed on normal page open | Passed. Success message was shown only after valid submit with success flag |
